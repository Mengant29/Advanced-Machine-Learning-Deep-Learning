Data preprocessing¶

The dataset has already exploitable values with no missing information, for instance. It's seems there is no need also to make relevant feature combinations: the various features seem completely independant. So all we did was normalizing the data with scaling. This step is only useful for models sensitive to scaling, like KNN.

Note: we do not change the format of the labels, keeping integer values between 1 and 10. We will see that it is necessary if we consider the problem as a regresion task. Also it is not penalizing if we consider it as a classification task on the other hand.

We use a min/max scaler since there are no outliers.

2. Model selection¶

We first chose to use reggresion models since this plot seems to indicate a gradient in the quality that seems even almost linear (quality accroding to volatile acidity and alcohol):

Since the mae or mse scoring is not relevant to compare classifiers and regressors, we chose to cross-validate on the predictive performances of each model. This means that we look at the cross-validation accuracy for the model selection.

The last plot above make us think that a simple linear regression would not have bad results since we have somehow a smooth gradient of the quality over some features (corresponding to the color gradient in the plotted figure).

A linear regression model will thus be a starting point that can then be used to evaluate the performance of the other more complex models we will use after.

Regressive models¶

  • With linear regression, we obtained a cross-validation accuracy of 0.58.
  • With SVM regressor, we obtained 0.61. We chose to try SVM since it is generally a performant model in general cases. But here, we found out that, with its default parameters, it performed just slightly better than the linear regression.
  • With Random forest regressor, we obtained 0.65. We chose to test a random forest because this model usually performs well in such machine learning tasks (Kaggle competitions). Here, it performs rather well compared to the linear regression, even though this accuracy is not that extraordinary. Nonetheless, our guess is that the underlying distribution data is so noisy that achieving way better accuracies is impossible.

Classification models¶

Maybe, regressive models are not as relevant as we first thought, so we tried to see the performances of classification models.

  • KNN may be a relevant classifier, though "expensive" in terms of prediction computing. We also need to estimate the right "k", the number of neighbors taken into account since it is the only important tunable parameter, and it si easy to see where is the best "k" by plotting the cross validation accuracy for a given range of values.
In [0]:
n_neighbors = np.arange(1, 50, 1)
accs = []
for n_neighbor in n_neighbors:
    model = KNeighborsClassifier(n_neighbors=n_neighbor, weights='distance')
    ypred = cross_val_predict(model, Xr_train, yr_train.ravel(), cv=4)
    accs.append(accuracy_score(ypred, yr_train))
plt.plot(n_neighbors, accs)
plt.xlabel("k (number of neighbors)")
plt.ylabel("cross validation accuracy")
plt.show()

We see that k is optimal around 15 with a cross validation accuracy of 0.65.

  • Random Forest classifier : we obtained a cross-validation accuracy of 0.64. Thus it achieves even better performances than its regressor version.

def rmse(y, y_hat):
  return np.sqrt(np.mean((y-y_hat)**2))

Conclusion¶

What we tried to have is a performant model rather than an explainable one. That is why we used a small set of rather simple models to select the one with the best performance. Then we tuned the hyperparameters to get the ones having the best cross-validation accuracy.
