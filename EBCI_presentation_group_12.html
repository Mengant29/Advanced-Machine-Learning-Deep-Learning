Introduction¶

Our approach was first to read the references about EEG signal classification, then fetch the different bits of codes and scripts to make the EEGNet_experimental baseline work.

Next, we used optuna to tune the some of the hyperparameters of the model.

Eventually, we considered the EEGNet as a feature extractor and use usual classifiers afterward.

In the end, we explored other techniques used in EEG signal classification.

Baseline model creation¶

The crossvalidation cv_per_subj_test function imported from the github notebook.

Important: We can see that 20% of the data was set aside for testing , with test_on_last_block = True. We also changed it a bit in terms of modularity with the parameter input write that indicates whether we want to write the results in files. We needed it in the hyperparameter tuning steps since the parallelism produced i/o errors, so it was better to rather make the function return those values instead of writing them in files.

def cv_per_subj_test(x, y, params, path_to_subj, test_on_last_block=False, plot_fold_history=False, write=False):
    if write:
        model_path = os.path.join(path_to_subj, 'checkpoints')
    else:
        model_path = None
    best_val_epochs = []
    best_val_aucs = []
    folds = 4  # To preserve split as 0.6 0.2 0.2
    if test_on_last_block:
        x_tr, y_tr, x_tst, y_tst = separte_last_block(x, y, test_size=0.2)
    else:
        x_tr, y_tr = x, y

    cv = StratifiedKFold(n_splits=folds, shuffle=True, random_state=0)
    cv_splits = list(cv.split(x_tr, y_tr))
    for fold, (train_idx, val_idx) in enumerate(cv_splits):
        if write:
            fold_model_path = os.path.join(model_path, '%d' % fold)
            os.makedirs(fold_model_path)
            model_path_ = os.path.join(fold_model_path, 'model{}'.format(fold))
        else:
            model_path_ = None
        x_tr_fold, y_tr_fold = x_tr[train_idx], y_tr[train_idx]
        x_val_fold, y_val_fold = x_tr[val_idx], y_tr[val_idx]

        val_history, fold_model = train_model_eegnet(x_tr_fold, y_tr_fold, params, (x_val_fold, y_val_fold), epochs=200,
                                                     batch_size=32, shuffle=True,
                                                     model_path=model_path_)
        best_val_epochs.append(np.argmax(val_history['val_auc']) + 1)  # epochs count from 1 (not from 0)
        best_val_aucs.append(np.max(val_history['val_auc']))
        if plot_fold_history:
            single_auc_loging(val_history, 'fold %d' % fold, fold_model_path)

    if test_on_last_block:
        test_history, final_model = train_model_eegnet(x_tr, y_tr, params, epochs=int(np.mean(best_val_epochs)),
                                                       validation_data=(x_tst, y_tst), batch_size=32, shuffle=True,
                                                       model_path=model_path)
        single_auc_loging(test_history, 'test_history', path_to_save=path_to_subj)
        
    if write:
        with codecs.open('%s/res.txt' % path_to_subj, 'w', encoding='utf8') as f:
            f.write(u'Val auc %.02f±%.02f\n' % (np.mean(best_val_aucs), np.std(best_val_aucs)))
            if test_on_last_block:
                f.write('Test auc naive %.02f\n' % (test_history['val_auc'][-1]))
                return test_history['val_auc'][-1], final_model
    
    return np.mean(best_val_aucs), np.std(best_val_aucs)

def train_cv(subjects, subjs_test_stats, test_on_last_block, params, write=False):
    subjs_test_stats = {}
    for train_subject in subjects.keys():
        if write:
            path_to_subj = prepare_dirs(experiment_res_dir, train_subject)
        else:
            path_to_subj = None
        x = subjects[train_subject][0]
        x = x.transpose(0, 2, 1)[:, np.newaxis, :, :]
        y = subjects[train_subject][1]
        test_auc, std_test_auc = cv_per_subj_test(x, y, params, path_to_subj, test_on_last_block=test_on_last_block, write=write)
        subjs_test_stats[train_subject] = (test_auc, std_test_auc)
    return subjs_test_stats

We chose to use Optuna in order to tune the parameters. The default/already tuned parameters were the following ones:

We eventually did not touch at 'D' and 'F1' because otherwise, the code was struggling to run for a reason we did not really understand. Also, we found out that 'norm_rate' was not used at all in the EEGNet model, so we removed it from the tuning.

We define the objective function that proceed to 1 trial, choosing randomly the values of the parameters according to the specification we set up, and returns the mean cross validation AUC.

In [5]:
def objective(trial):

    resample_val = 
    custom_params = {}
    
    # fixed parameters
    custom_params['D'] = 3
    custom_params['F1'] = 12
    custom_params['norm_rate'] = 0.275
    
    # tunable parameters
    custom_params['resample_to'] = trial.suggest_int('resample_to', 380, 420)
    custom_params['dropoutRate1'] = trial.suggest_uniform('dropoutRate1', 0.2, 0.8)
    custom_params['dropoutRate2'] = trial.suggest_uniform('dropoutRate2', 0.2, 0.8)
    custom_params['lr'] = trial.suggest_loguniform('lr', 1e-5, 1e-4)
    
    subjects = createSubjectsWithResample(resample_val)
    subjs_test_stats = {}
    mean_AUCs = train_cv(subjects, subjs_test_stats, test_on_last_block=False, params=custom_params)    
    
    return mean_AUCs

We then start the optimization by collecting the AUCs obtained for each set of parameters. We run it for at most 3 hours (due to automated execution termination constraints), with a maximum of 100 trials (that will not be reached in such time constraints).

 Extension: Changing the classifier¶

We consider the ouput of forward_features() as the input of a classifier. In the pipeline, we train the EEGNet as usual on the training data, then, we hope that the forward_features() extract well enough the features so that we use usual classifiers on it.

As we want to use cross validation to evaluate our model, and we do not want to train the EEGNet everytimes, we chose to fix the random_state of the StratifiedKFold object to have the same train/validation splits for preatrained models (for which the weights just need to be loaded), and for classifiers.

Note: We first performed a CV grid search with fixed extracted features from a model trained on the train set. But the problem is that even though the classifier does not learn from the validation data, the way the features are extracted has been learnt from the whole training data before the splitting. It means that the features extraction is already adapted to the cross validation data at every split. The classfier then easily reaches better performances than the complete EEGNet, but it is cheating somehow.

model_cv() solves this problem by loading the weights model trained given a certain split, and returning the output of forward_features() from both the training and validation data, and also returns the prediction of the complete EEGNet model on the unseen validation data:

In [29]:
def model_cv(subj, x, y, cv_num):
    model_dict = torch.load(os.path.join("./res", str(subj), "checkpoints", str(cv_num), "model{}".format(cv_num)))
    kernLength1 = best_params.get('kernLength1',64)
    kernLength2 = best_params.get('kernLength2',16)
    F2 = best_params.get('F2',best_params['F1']*best_params['D'])

    model = EEGNet_experimental(nb_classes=2, Chans=x.shape[2], Samples=x.shape[3],
                                   dropoutRates=(best_params['dropoutRate1'], best_params['dropoutRate2']),
                                   kernLength1=kernLength1, kernLength2=kernLength2, poolKern1=4,
                                   poolKern2=8,
                                   F1=best_params['F1'],
                                   D=best_params['D'], F2=F2)

    # using gpu if possible (as we are not going to train anymore, but just predict, not very important)
    device = torch.device('cuda:0' if torch.cuda.is_available() else 'cpu')
    model.to(device)
    
    skf = StratifiedKFold(n_splits=4, shuffle=True, random_state=0)
    train_idx, val_idx = list(skf.split(x, y))[cv_num]
    
    x_tensor = torch.from_numpy(x[train_idx]).float().to(device)
    x_val_tensor = torch.from_numpy(x[val_idx]).float().to(device)
    model.load_state_dict(model_dict)
    
    with torch.no_grad():
        model.eval()
        features, _ = model.forward_features(x_tensor)
        features_val, _ = model.forward_features(x_val_tensor)
        y_full = torch.nn.Softmax(dim=1)(torch.sigmoid(model.forward(x_val_tensor)))[:, 1]
    return model.flatten(features).numpy(), model.flatten(features_val).numpy(), y_full.numpy()

Given a classifier, compare_cv() will return the AUC means and stds of the classifier vs complete EEGNet:

In [30]:
def compare_cv(clf, x, y, n_split=4):
    
    auc_clf = []
    auc_full = []
    for cv in range(n_split):
        features, features_val, y_full_forward = model_cv(25, x, y, cv)
                
        skf = StratifiedKFold(n_splits=n_split, shuffle=True, random_state=0)
        train_idx, val_idx = list(skf.split(x, y))[cv]
        
        clf.fit(features, y[train_idx])
        
        auc_clf.append(roc_auc_score(y[val_idx], clf.predict(features_val)))
        auc_full.append(roc_auc_score(y[val_idx], y_full_forward))
    auc_clf = np.array(auc_clf)
    auc_full = np.array(auc_full)
    
    return (auc_clf.mean(), auc_clf.std()), (auc_full.mean(), auc_full.std())

It seems that the feature extraction proceeded by an EEGNet si only exploitable for itself. As for the other classifiers, SVM seems to be the best one (even though its AUC is far behind the EEGNet one). The conclusion with these results is that the vanilla pipeline of EEGNet seems to be the most performant, and that is the one we should consider.

The two approaches produce similar results that are not as satisfying as EEGNet (respectively 0.71 and 0.93). In fact, it seems to be a classical/basic approach as for EEG signal classification. EEGNet is more complex, and turns out to suit well to this problem. But still, the Riemannian geometry models are way faster to compute, and easy to setup.

Note: Once again, we only worked with one subject (25), so it would be preferable to have the results with all the subjects before totally giving up the Riemannian geometry approach.
